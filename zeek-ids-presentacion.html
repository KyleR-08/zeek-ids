Introduction

What is Zeek?

📡

Network Analysis Framework

Open source tool for deep monitoring and analysis of network traffic. Previously known as "Bro".

📝

Own Scripting Language

Zeek has its own event-oriented scripting language — you define exactly what to detect and how to report it.

🔍

IDS — Detection Only

Passively observes traffic. Generates alerts but does not block connections — that would be an IPS.

📊

Detailed Logging

Automatically logs connections, DNS, HTTP, SSL, FTP and more into structured log files.

🔬

.pcap File Analysis

Can analyze network traffic recordings to investigate past incidents — forensic analysis.

⚙️

Version 8.2.0

Installed on Ubuntu 24.04 over WSL2 on Windows 11. Official OpenSUSE repository.

📄 ids_main.zeek

Entry point script. Uses @load to import modules and defines zeek_init and zeek_done events that display the start and end banners.

🚫 blacklist.zeek

Detection module. Defines the IP blacklist and the new_connection event that compares each connection against the list.

Python script using Scapy that builds artificial TCP packets toward blacklisted IPs and saves them into a .pcap file for testing.

Implementation

The Code Explained

blacklist.zeek
# Malicious IP address list
global blacklist: set[addr] = {
  1.2.3.4, 5.6.7.8, 10.0.0.99,
  192.168.1.254, 45.33.32.156,
  198.20.99.130
};

event new_connection(c: connection)
  {
  local dst = c$id$resp_h; // destination IP
  local src = c$id$orig_h; // source IP

  if ( dst in blacklist )
    {
    print fmt("[ALERT] %s → %s",
      src, dst);
    }
  }
①
global blacklist: set[addr]

Global variable of type set of IP addresses. A set allows instant lookup — ideal for matching thousands of IPs per second.

②
event new_connection

Zeek calls this function automatically every time it detects a new TCP connection. Parameter c holds all connection details.

③
c$id$resp_h / c$id$orig_h

Access connection object fields using Zeek's $ operator. resp_h = destination IP, orig_h = source IP.

④
if dst in blacklist

Checks if the destination IP is in the set. If it matches, fmt() formats the message and print outputs the alert.

Workflow

How to Use It

01

Clone the repository

Download the project from GitHub with all scripts included.

git clone https://github.com/KyleR-08/zeek-ids.git && cd zeek-ids
02

Install Python dependency

Scapy is required to run the attack traffic generator script.

pip3 install scapy --break-system-packages
03

Generate simulated attack traffic

The Python script creates 81 TCP packets with connections to blacklisted IPs, saved as ataques_simulados.pcap.

python3 generar_ataques.py
04

Run the IDS

Zeek analyzes the .pcap with our scripts. The -C flag ignores WSL checksum errors. Alerts appear in real time.

zeek -r ataques_simulados.pcap ids_main.zeek -C

Why Zeek?

Zeek allows writing completely custom detection logic from scratch, making it ideal for understanding how an IDS works internally. Its event engine gave us full control over what to analyze and how to report it.

Closing

Conclusions

  • 01Zeek is more than a traditional IDS — it is a platform that allows building fully custom detection logic through its own network event-oriented scripting language.
  • 02We implemented a functional IDS that detects connections to malicious IPs in real time, analyzing traffic from .pcap files generated with Scapy.
  • 03Zeek's new_connection event lets us intercept every TCP connection and compare the destination IP against our blacklist in milliseconds.
